📘 HTML Master Reference

This page is a complete reference for learning HTML — from absolute basics to advanced topics.

🚀 Introduction & What You Need to Learn

HTML stands for HyperText Markup Language. It is the core building block of the web, used to structure and present content in browsers.

  • 🧠 HTML is a markup language, not a programming language.
  • 🏗️ It defines the structure of web pages using elements called tags.
  • 🌐 HTML elements represent text, links, images, multimedia, forms, and layout blocks.

🎯 What You Will Learn

  • Basic structure of an HTML document
  • Using tags, attributes, and elements
  • Working with text, lists, links, images, and tables
  • Creating forms and interactive elements
  • Building semantic, accessible, and responsive layouts

🌐 HTML Elements & How the Browser Renders Them

HTML is made up of elements — which are defined using tags. Tags tell the browser how to display the content between them.

📦 Example of an Element:

<p>This is a paragraph</p>

In this example:

  • <p> is the opening tag
  • This is a paragraph is the content
  • </p> is the closing tag

🔁 Nesting Elements

HTML elements can be nested inside each other. For example: 


      <div>
        <h2>Title</h2>
        <p>Some text here.</p>
      </div>

👁️ Browser Behavior

  • Browsers ignore extra white space and line breaks between elements.
  • HTML is not case-sensitive — but lowercase is recommended.
  • Browsers build a DOM (Document Object Model) tree based on elements.

✅ Best Practices:

  • Always use lowercase tags: <body> not <BODY>
  • Indent nested tags for better readability
  • Close all tags properly

🧱 The <head>, <body>, and Nested Elements

Every HTML page consists of two main sections:

  • <head>: Contains metadata, title, links to styles, fonts, and scripts. It’s not visible in the browser window.
  • <body>: Contains all the visible content of the webpage (text, images, links, etc.).

🧩 Example HTML Structure:


      <!DOCTYPE html>
      <html lang="en">
        <head>
          <meta charset="UTF-8">
          <title>My Webpage</title>
        </head>
        <body>
          <h1>Welcome!</h1>
          <p>This is a simple HTML structure.</p>
        </body>
      </html>

🔁 Nesting Tags Properly

  • Tags must be properly nested and closed in the correct order.
  • Incorrect: <p><strong>Bold</p></strong>
  • Correct: <p><strong>Bold</strong></p>

🧠 Browser Rendering Flow

  • The browser starts parsing from the top: it sees <!DOCTYPE html> and enters Standards Mode.
  • It reads <html>, then <head> for settings, then finally renders the <body>.

🔠 Headings and Use Cases

Headings help structure your content semantically. They also help search engines and screen readers understand your page hierarchy.

📚 Six Levels of Headings:

  • <h1> – Main title (used only once per page)
  • <h2> – Section title
  • <h3> – Subsection under h2
  • <h4> – Sub-subsection
  • <h5> – Very detailed section title
  • <h6> – The smallest and least used heading

✅ Best Practices:

  • Use only one <h1> per page (represents the main topic)
  • Do not skip levels (e.g., avoid <h1> → <h4>)
  • Use headings in order to create a meaningful structure

📦 Example:


      <h1>HTML Reference Guide</h1>
        <h2>Introduction</h2>
          <h3>What is HTML?</h3>
        <h2>Main Topics</h2>
          <h3>Elements</h3>
          <h3>Attributes</h3>

🎯 Tip:

Search engines use <h1> to <h3> to index the main sections of your site.

🧪 Syntax Rules & Validation

Writing correct HTML syntax helps ensure that all browsers interpret your page consistently and without errors.

📐 Syntax Rules:

  • Tags should always be enclosed in angle brackets: <tag>
  • Most elements require an opening and closing tag: <p>...</p>
  • Self-closing tags end with a slash: <br />, <img />
  • Attributes go inside the opening tag: <a href="https://example.com">
  • Attribute values should be in double quotes: "value"

✅ Valid Example:

<a href="https://example.com" target="_blank">Visit</a>

❌ Invalid Example:

<a href=https://example.com>Visit</a>

🧪 Testing & Validation Tools:

  • W3C Markup Validation Service
  • HTMLHint – Linter for your HTML code
  • Use browser developer tools (F12) to inspect and debug HTML structure

🧠 Tip:

Use a code editor like Visual Studio Code with HTML extensions to get real-time validation and autocomplete.

📄 The <p> Element

The <p> tag defines a paragraph of text. It is a block-level element that adds vertical spacing before and after the content.

🎯 Use Cases:

  • Organize text into readable sections
  • Separate thoughts or ideas in content
  • Used inside other block elements like <article>, <section>, and <div>

✅ Example:

<p>This is a simple paragraph in HTML.</p>

🧠 Behavior Notes:

  • Browsers automatically add margin above and below each paragraph
  • It can contain inline elements like <strong>, <em>, <a>

🔗 Example with inline formatting:

<p>
        This is a <strong>bold</strong> word and this is an <em>italic</em> one.
      </p>

🚫 Don't Do This:

<p>Paragraph 1</p> <p>Paragraph 2</p>

✳️ Don't nest <p> inside another <p>; that is invalid HTML.

🏷️ HTML Attributes

Attributes provide additional information about HTML elements. They're always written in the opening tag as name="value" pairs.

📌 Syntax:

<element attribute="value">Content</element>

🔹 Example:

<img src="image.jpg" alt="A sample image" width="300" />

🧩 Global Attributes (can be used on most elements):

  • id: Unique identifier
  • class: Used for styling with CSS or JS targeting
  • style: Inline styling
  • title: Tooltip on hover
  • hidden: Hides the element
  • tabindex: Controls tab order
  • accesskey: Sets a keyboard shortcut
  • draggable: Allows dragging

💡 Example with multiple attributes:

<p id="intro" class="lead" title="Hover text">
        Welcome to HTML attributes.
      </p>

🛠️ Specific Attributes (for certain elements):

  • href – for <a>
  • src, alt, width – for <img>
  • type, value, placeholder – for <input>

🔐 Boolean Attributes:

These attributes don’t need a value. If present, they're treated as true. 

<input type="checkbox" checked />
      <input type="text" disabled />

🎯 Tips:

  • Always wrap attribute values in double quotes
  • Use lowercase for attribute names
  • Don’t duplicate attributes on the same element

🎨 Text Formatting Elements

HTML provides several inline elements that help format and style text without using CSS. These tags describe the meaning or importance of the content.

🧩 Common Formatting Tags:

  • <strong> – Important text (usually bold)
  • <em> – Emphasized text (usually italic)
  • <b> – Bold (without semantic meaning)
  • <i> – Italic (without semantic meaning)
  • <mark> – Highlighted text
  • <small> – Smaller text
  • <del> – Deleted text (strikethrough)
  • <ins> – Inserted text (usually underlined)
  • <sub> – Subscript (e.g. H2O)
  • <sup> – Superscript (e.g. x2)

✅ Examples:

<p>This is <strong>important</strong> information.</p>
      <p>This is <em>emphasized</em> content.</p>
      <p>Price: <del>$50</del> <ins>$30</ins></p>
      <p>Water formula: H<sub>2</sub>O</p>
      <p>Math: x<sup>2</sup></p>
      <p><mark>Highlighted</mark> word</p>

🧠 Notes:

  • <strong> and <em> are better for accessibility than <b> and <i>
  • Use <mark> to highlight search terms
  • <del> and <ins> are useful in content edits and versioning

🔗 Links and the <a> Tag

The <a> (anchor) tag is used to create hyperlinks that navigate users to other pages, files, sections, or websites.

🔹 Basic Syntax:

<a href="URL">Link Text</a>

🌍 Example (External Link):

<a href="https://www.google.com">Go to Google</a>

📂 Example (Internal Link to Another Page):

<a href="about.html">About Us</a>

📌 Anchor Tag Attributes:

  • href: Required. Specifies the destination URL.
  • target: Where to open the link:
    • _self: Default, opens in the same tab
    • _blank: Opens in a new tab 🔗
  • title: Tooltip text on hover
  • download: Initiates file download

🧩 Example with `target="_blank"` and title:

<a href="https://developer.mozilla.org" target="_blank" title="MDN Docs">MDN</a>

📍 Internal Section Link (Page Navigation):


      <a href="#section2">Jump to Section 2</a>

      ...

      <h2 id="section2">Section 2</h2>

💾 Download File Link:

<a href="files/manual.pdf" download>Download Manual</a>

🔐 Email Link (mailto:):

<a href="mailto:example@email.com">Send Email</a>

📞 Phone Link (tel:):

<a href="tel:+201001234567">Call Us</a>

⚠️ Accessibility Tip:

Make link text meaningful like Visit our contact page instead of just Click here.

🖼️ Images and the <img> Tag

The <img> tag is used to display images in a webpage. It's a self-closing tag and must include the src and alt attributes.

🔹 Syntax:

<img src="image.jpg" alt="Description of the image" />

📍 Required Attributes:

  • src – Path to the image file (relative or absolute)
  • alt – Descriptive text for screen readers or when the image fails to load

📁 Path Types:

  • Relative Path: Refers to a file in the same project folder
    src="images/pic.jpg"
  • Absolute Path: Full URL or full local path
    src="https://example.com/images/pic.jpg"

📷 Example 1: Internal Image

<img src="logo.png" alt="Website Logo" width="200" />

🌍 Example 2: External Image

<img src="https://cdn-icons-png.flaticon.com/512/25/25231.png" alt="GitHub Logo" width="100" />

⚠️ Best Practices:

  • Use descriptive and meaningful alt text for accessibility.
  • Use optimized image formats like .webp or compressed .jpg/.png.
  • Avoid using images with large file sizes to maintain fast loading speed.
  • Add width or height to control display size.

🧠 Tips:

  • If the image is for decoration only, you can leave alt="" empty.
  • For logos, always describe the brand or logo name in alt.

🛑 Common Mistake:

<img> ← Missing "src" and "alt"

📝 Lists in HTML

HTML offers three main types of lists to organize and display grouped items:

1️⃣ Unordered List - <ul>

Displays items with bullets. Use when order doesn’t matter.

<ul>
        <li>HTML</li>
        <li>CSS</li>
        <li>JavaScript</li>
      </ul>

2️⃣ Ordered List - <ol>

Displays items with numbers. Use when the order is important.

<ol>
        <li>Wake up</li>
        <li>Brush teeth</li>
        <li>Start coding</li>
      </ol>

🔸 Type Attribute (for <ol>):

  • type="1" – 1, 2, 3… (default)
  • type="A" – A, B, C…
  • type="a" – a, b, c…
  • type="I" – I, II, III…
  • type="i" – i, ii, iii…
<ol type="A">
        <li>Step A</li>
        <li>Step B</li>
      </ol>

3️⃣ Description List - <dl>

Used to pair terms and their descriptions.

<dl>
        <dt>HTML</dt>
        <dd>HyperText Markup Language</dd>
        <dt>CSS</dt>
        <dd>Cascading Style Sheets</dd>
      </dl>

🔁 Nested Lists:

<ul>
        <li>Frontend
          <ul>
            <li>HTML</li>
            <li>CSS</li>
          </ul>
        </li>
        <li>Backend</li>
      </ul>

✅ Best Practices:

  • Use semantic list types for proper structure
  • Use nested lists only when necessary
  • Never mix text and lists on the same level without using <li>

🧠 Accessibility Tip:

Screen readers rely on correct list structures to communicate grouped content clearly.

📊 Tables in HTML

Tables are used to organize and display tabular data using rows and columns.

📐 Table Structure:

<table>
        <thead>
          <tr>
            <th>Name</th>
            <th>Age</th>
          </tr>
        </thead>
        <tbody>
          <tr>
            <td>Alice</td>
            <td>25</td>
          </tr>
          <tr>
            <td>Bob</td>
            <td>30</td>
          </tr>
        </tbody>
      </table>

🔹 Tags Overview:

  • <table> – The entire table container
  • <thead> – Header section of the table
  • <tbody> – Body section containing data rows
  • <tfoot> – Footer section (optional)
  • <tr> – Table row
  • <th> – Table header cell (bold, centered by default)
  • <td> – Table data cell

🎨 Adding Borders (CSS Example):

<style>
        table {
          border-collapse: collapse;
          width: 100%;
        }
        th, td {
          border: 1px solid #ccc;
          padding: 10px;
        }
        thead {
          background-color: #eee;
        }
      </style>

📌 Example with Styling:

<table>
        <thead>
          <tr>
            <th>Language</th>
            <th>Year Created</th>
          </tr>
        </thead>
        <tbody>
          <tr>
            <td>HTML</td>
            <td>1993</td>
          </tr>
          <tr>
            <td>CSS</td>
            <td>1996</td>
          </tr>
        </tbody>
      </table>

➕ Optional Attributes:

  • colspan – Merge cells across columns
  • rowspan – Merge cells across rows
<td colspan="2">Merged Cell</td>

✅ Best Practices:

  • Use <thead> and <th> for semantic clarity
  • Use CSS for styling (avoid HTML border attribute)
  • Add scope="col" or scope="row" for accessibility

🧩 Span, Break, and Horizontal Rule

🔹 1. <span> – Inline Container

The <span> tag is an inline element used to apply styles or scripts to a part of text. 

<p>
        This is a <span style="color: red;">highlighted</span> word.
      </p>
  • 🧠 Used with CSS or JavaScript to target specific words.
  • Does not add any visual change by default.
---

🔹 2. <br> – Line Break

The <br> tag creates a single line break. It's a self-closing tag. 

<p>Line one<br>Line two</p>
  • ✅ Use inside paragraphs or text when you need a manual line break.
  • ⚠️ Do not use multiple <br> tags for spacing — use CSS instead.
---

🔹 3. <hr> – Horizontal Rule

The <hr> tag creates a horizontal line and separates content. It’s also a self-closing tag. 

<h2>Section 1</h2>
      <p>Some content</p>
      <hr>
      <h2>Section 2</h2>
      <p>More content</p>
  • Used to indicate thematic breaks or section changes.
  • Can be styled with CSS to change its appearance.
<style>
        hr {
          border: none;
          height: 2px;
          background-color: #888;
          margin: 2em 0;
        }
      </style>

📦 Using <div> to Organize Layout

The <div> (short for "division") is a block-level container used to group related elements together. 

<div>
        <h3>User Profile</h3>
        <p>Name: Alice</p>
        <p>Age: 25</p>
      </div>

🔹 Why Use `
`?

  • ✅ Structure your HTML layout
  • ✅ Apply CSS styles to entire sections
  • ✅ Add JavaScript interactivity

🎨 CSS Example with `class`:

<style>
        .card {
          border: 1px solid #ccc;
          padding: 15px;
          margin-bottom: 20px;
          border-radius: 8px;
          background-color: #f9f9f9;
        }
      </style>

      <div class="card">
        <h4>Article Title</h4>
        <p>Some description here...</p>
      </div>

🧠 Best Practices:

  • Use <div> when no semantic tag is available.
  • Add class or id to apply styles or behaviors.
  • Too many divs can create “div soup” — prefer semantic tags when possible.

📌 Example: Layout with Divs

<div class="header">Header Section</div>
      <div class="main">Main Content</div>
      <div class="footer">Footer Section</div>

🔣 HTML Entities

HTML entities are used to display reserved characters (like < or &) or special symbols (like © or €).

🔹 Why Use Entities?

  • Some characters are reserved in HTML, like < and >.
  • Entities let you show characters that would otherwise be treated as code.
  • You can also insert invisible characters (e.g., spaces) or non-keyboard symbols.

🔸 Common Entities:

Character Entity Description
< &lt; Less than
> &gt; Greater than
& &amp; Ampersand
" &quot; Double Quote
' &apos; Single Quote (not in HTML4)
© &copy; Copyright Symbol
€ &euro; Euro Sign
  &nbsp; Non-breaking Space

🧪 Example:

<p>5 &lt; 10 is a valid math expression.</p>
      <p>Use &copy; 2025 for copyright notice.</p>
      <p>Multiple &nbsp;&nbsp;&nbsp; spaces</p>

🔍 Output:

5 < 10 is a valid math expression.

Use © 2025 for copyright notice.

Multiple    spaces

🏗️ Semantic HTML Elements

Semantic elements clearly describe their meaning to both the browser and developers.

🔹 Why Use Semantic Tags?

  • ✅ Improve readability and code structure
  • ✅ Help search engines understand your content (SEO)
  • ✅ Support accessibility tools like screen readers

🔸 Common Semantic Tags:

  • <header>: Top section (logo, nav, etc.)
  • <nav>: Navigation links
  • <main>: Main content area
  • <section>: Logical sections or topics
  • <article>: Independent, self-contained content (e.g., blog post)
  • <aside>: Sidebar or related content
  • <footer>: Bottom section (copyright, contact)

📌 Example Layout:

<header>
        <h1>My Website</h1>
        <nav>
          <a href="#">Home</a>
          <a href="#">Blog</a>
          <a href="#">Contact</a>
        </nav>
      </header>

      <main>
        <section>
          <h2>About Us</h2>
          <p>We build modern websites.</p>
        </section>

        <article>
          <h2>Latest News</h2>
          <p>HTML6 coming soon!</p>
        </article>

        <aside>
          <p>Subscribe to our newsletter</p>
        </aside>
      </main>

      <footer>
        <p>© 2025 My Company</p>
      </footer>

🧠 Best Practices:

  • Use semantic elements instead of generic <div> where applicable.
  • Don't use semantic tags just for styling — use them for structure and meaning.

📐 Layout with <div> and Classes

The <div> element is a generic container for grouping content. It has no semantic meaning, but it is extremely useful for layout and styling.

🔹 When to Use <div>

  • When semantic tags like <header>, <main>, etc. are not appropriate.
  • To group elements logically and apply common styles using CSS classes or IDs.
  • To structure layouts (like grids, cards, columns, containers).

🔸 Example:

<div class="container">
        <div class="card">
          <h3>Card Title</h3>
          <p>Some descriptive text inside the card.</p>
        </div>

        <div class="card">
          <h3>Another Card</h3>
          <p>More content here.</p>
        </div>
      </div>

💅 Sample CSS (in <style> block):

/* This CSS goes inside the <style> tag in <head> */
      .container {
        display: flex;
        gap: 20px;
        flex-wrap: wrap;
        padding: 20px;
      }
      .card {
        background-color: #f0f0f0;
        padding: 15px;
        border-radius: 10px;
        width: 300px;
        box-shadow: 0 0 5px #ccc;
      }

🧠 Best Practices:

  • Use class names that describe the content or purpose, like card, nav-container, hero-section, etc.
  • Try to use semantic tags first, and fall back to <div> only when needed.

🏗️ Layout with Semantic HTML

Instead of using generic <div> tags everywhere, HTML5 introduced semantic tags that give meaning to different parts of a page.

🔹 Benefits of Semantic Layout:

  • Improved accessibility (screen readers understand content better)
  • Better SEO (search engines parse page meaning)
  • Easier to maintain and read

🔸 Common Layout Tags:

  • <header> – Website or section header
  • <nav> – Navigation menus
  • <main> – Central content area
  • <section> – Group of related content
  • <article> – Independent piece of content (e.g. blog post)
  • <aside> – Sidebar or secondary content
  • <footer> – Bottom section of page or section

📐 Semantic Page Structure Example:

<header>
        <h1>My Blog</h1>
        <nav>
          <a href="#">Home</a>
          <a href="#">Posts</a>
          <a href="#">About</a>
        </nav>
      </header>

      <main>
        <section>
          <article>
            <h2>Post Title</h2>
            <p>Lorem ipsum content for the article.</p>
          </article>

          <article>
            <h2>Second Post</h2>
            <p>Another piece of awesome content!</p>
          </article>
        </section>

        <aside>
          <h3>About Me</h3>
          <p>Web developer & content creator.</p>
        </aside>
      </main>

      <footer>
        <p>© 2025 My Blog. All rights reserved.</p>
      </footer>

💡 Tip:

  • You can use semantic tags together with classes and CSS just like you would with divs.
  • Keep your structure logical and consistent for better accessibility and maintenance.

🎵 The <audio> Element

The <audio> tag allows you to embed audio files like music, sound effects, or voiceovers into your web page.

🔹 Basic Syntax:

<audio src="audio-file.mp3" controls></audio>

🔸 Common Attributes:

  • src: Path to the audio file (MP3, OGG, WAV)
  • controls: Displays play/pause, volume, etc.
  • autoplay: Starts playing automatically (often blocked by browsers)
  • loop: Repeats the audio continuously
  • muted: Starts muted

🔸 Multiple Source Example (for browser compatibility):

<audio controls>
        <source src="audio.mp3" type="audio/mpeg" />
        <source src="audio.ogg" type="audio/ogg" />
        Your browser does not support the audio tag.
      </audio>

✅ Best Practices:

  • Include fallback text for unsupported browsers.
  • Use multiple formats (MP3, OGG) if needed for compatibility.
  • Do not autoplay audio unless necessary and silent.

🎬 The <video> Element

The <video> tag lets you embed videos into your web page. It's great for tutorials, backgrounds, or media content.

🔹 Basic Syntax:

<video src="video.mp4" controls></video>

🔸 Common Attributes:

  • src: Path to the video file (MP4, WebM, OGG)
  • controls: Shows play, pause, volume, etc.
  • autoplay: Automatically plays on load (must be muted in many browsers)
  • loop: Restarts automatically when finished
  • muted: Starts with no sound
  • poster: Image shown before the video starts
  • width and height: Set dimensions

🔸 Example with Multiple Sources:

<video controls width="400" poster="preview.jpg">
        <source src="video.mp4" type="video/mp4" />
        <source src="video.webm" type="video/webm" />
        Your browser does not support the video tag.
      </video>

🎯 Use Cases:

  • Tutorials
  • Marketing or product showcases
  • Background videos (with muted + autoplay)

💡 Best Practices:

  • Compress videos for faster load times.
  • Use the poster attribute to avoid blank screens.
  • Always provide controls unless the video is decorative.

🧾 Forms in HTML - Part 1

HTML forms are used to collect input from users. They’re essential for features like login, signup, search, surveys, and file uploads.

🔹 Basic Form Structure:

<form action="submit.php" method="POST">
        <!-- form elements here -->
      </form>

🔸 Important Attributes:

  • action: Where to send form data (URL or script)
  • method: HTTP method (GET or POST)
  • name: Identifier for the form
  • target: Where to display the result (e.g. _blank)
  • novalidate: Disables HTML5 validation

🔸 Example:

<form action="/submit" method="POST">
        <label for="username">Username:</label>
        <input type="text" id="username" name="username" required />
        <button type="submit">Send</button>
      </form>

📥 Input Types:

Each <input> element can have different type depending on the kind of data you need:

  • text: Regular text
  • email: Must be a valid email
  • password: Obscured input
  • number: Numeric input
  • range: Slider for a numeric range
  • color: Color picker
  • date, time, datetime-local
  • url: Must be a valid link
  • search: Styled search field
  • tel: Telephone number
  • file: Upload a file
  • hidden: Hidden value
  • checkbox: On/off toggle
  • radio: Choose one from multiple
  • submit, reset, button

📌 Example with Various Inputs:

<form>
        <label>Name:</label>
        <input type="text" name="name" required /><br />

        <label>Age:</label>
        <input type="number" name="age" /><br />

        <label>Email:</label>
        <input type="email" name="email" required /><br />

        <label>Choose Color:</label>
        <input type="color" name="favcolor" /><br />

        <label>Upload CV:</label>
        <input type="file" name="cv" /><br />

        <label>Subscribe:</label>
        <input type="checkbox" name="subscribe" /><br />

        <button type="submit">Submit</button>
      </form>

🧾 HTML Forms - Part 2

📝 Textarea (Multi-line Text Input)

The <textarea> is used for large blocks of user input like comments or messages.

<textarea name="message" rows="5" cols="30">
      Write here...
      </textarea>

🔽 Select Menu (Dropdown)

Use <select> with <option> to create dropdowns.

<label for="car">Choose a car:</label>
      <select id="car" name="car">
        <option value="volvo">Volvo</option>
        <option value="bmw">BMW</option>
        <option value="audi">Audi</option>
      </select>

🎯 Radio Buttons

Use the same name for a group to make them exclusive.


      <p>Gender:</p>
      <input type="radio" id="male" name="gender" value="male" />
      <label for="male">Male</label>

      <input type="radio" id="female" name="gender" value="female" />
      <label for="female">Female</label>

✅ Checkboxes

Each checkbox is independent.


      <p>Skills:</p>
      <input type="checkbox" id="html" name="skills" value="HTML" />
      <label for="html">HTML</label>

      <input type="checkbox" id="css" name="skills" value="CSS" />
      <label for="css">CSS</label>

📋 Datalist (Auto-suggestions)

<input list="browsers" name="browser" />
      <datalist id="browsers">
        <option value="Chrome" />
        <option value="Firefox" />
        <option value="Edge" />
        <option value="Safari" />
      </datalist>

🔐 Special Attributes:

  • readonly: Field cannot be modified
  • disabled: Field is not editable or submitted
  • autofocus: Auto-focus when page loads
  • placeholder: Hint text in input field
  • required: Must be filled out before submitting
  • value: Default value

🔘 Reset & Submit Buttons


      <input type="submit" value="Send" />
      <input type="reset" value="Reset" />

📁 File Upload


      <form action="/upload" method="post" enctype="multipart/form-data">
        <input type="file" name="resume" />
        <button type="submit">Upload</button>
      </form>

🕒 Time & Date Inputs


      <input type="date" name="dob" />
      <input type="time" name="alarm" />
      <input type="datetime-local" name="meeting" />

🔒 Hidden Input

Used to store extra data sent with the form:

<input type="hidden" name="userId" value="12345" />

📌 Special HTML Elements

🗨️ <blockquote>

Used for long quotations. It usually adds indentation to the text to visually indicate a quote. 

<blockquote cite="https://en.wikipedia.org/wiki/HTML">
        HTML is the standard markup language for documents designed to be displayed in a web browser.
      </blockquote>

📎 cite attribute

Specifies the source URL of the quotation.

🔁 <wbr>

<wbr> stands for “word break opportunity”. It defines a spot where the browser may optionally break a line.

Use it for long strings like URLs or emails to allow better wrapping:

<p>
        https://www.verylong<wbr>urlwithnobreak<wbr>available.com
      </p>

🌍 <bdi> (Bi-Directional Isolation)

Use <bdi> when you want to isolate part of the text with unknown direction (LTR or RTL). 

<p>Usernames:
        <bdi>جون</bdi>
        <bdi>James</bdi>
      </p>

This prevents layout issues when mixing Arabic, English, or Hebrew content.

🔘 <button>

The <button> element creates a clickable button.

🔹 Types:

  • type="submit": Submits the form (default inside forms)
  • type="reset": Resets all form inputs
  • type="button": Regular button (used with JS)
<button type="submit">Submit Form</button>
      <button type="reset">Reset All</button>
      <button type="button" onclick="alert('Clicked!')">Click Me</button>

You can also use icons and images inside buttons:

<button>
        <img src="icon.png" width="16" /> Download
      </button>

🧱 Embedding & Code Display Elements

🖼️ <iframe> (Inline Frame)

<iframe> is used to embed another webpage within the current page. 

<iframe src="https://www.example.com" width="500" height="300"></iframe>

🔐 Security Notes

  • Use sandbox to limit iframe capabilities
  • Use loading="lazy" to defer loading
<iframe
        src="https://elzero.org"
        width="600"
        height="300"
        sandbox
        loading="lazy"
      ></iframe>

⏹️ <pre> (Preformatted Text)

<pre> preserves whitespace, line breaks, and formatting exactly as written in the HTML. 

<pre>
      Line 1
        Line 2
          Line 3
      </pre>

💻 <code> (Inline Code)

Used to display inline programming code or tags. 

<p>Use the <code><img></code> tag to add images.</p>

👯 Combine with <pre>

You can use <pre> + <code> to format blocks of code: 

<pre><code>
      function greet() {
        alert("Hello!");
      }
      greet();
      </code></pre>

♿ Accessibility & ARIA

Accessibility means making websites usable for everyone, including people with disabilities.

🔍 Why Accessibility Matters?

  • Helps screen readers interpret content correctly
  • Makes sites usable with keyboards (tab, enter, etc.)
  • Improves SEO and usability for all users

🔖 Semantic HTML = Built-in Accessibility

Use elements like <header>, <nav>, <main>, <article>, etc. instead of <div> to give meaning to content.

✅ Example:

<nav>
        <ul>
          <li><a href="index.html">Home</a></li>
          <li><a href="about.html">About</a></li>
        </ul>
      </nav>

📚 ARIA (Accessible Rich Internet Applications)

ARIA attributes help make non-semantic elements accessible (like buttons built from <div>).

⚠️ Only use ARIA if there’s no native HTML alternative!

✅ Common ARIA Attributes:

  • role – defines what the element is
  • aria-label – adds a label for screen readers
  • aria-hidden="true" – hides from screen readers

📌 Example:

<div role="button" tabindex="0" aria-label="Open Menu">
        ☰
      </div>

🧭 Keyboard Navigation

Use tabindex="0" to make elements focusable by keyboard.

<div tabindex="0">I can be focused with the Tab key</div>

🗣️ Screen Reader Text

Use a visually hidden label for screen readers:

<style>
      .visually-hidden {
        position: absolute;
        width: 1px;
        height: 1px;
        margin: -1px;
        overflow: hidden;
        clip: rect(0, 0, 0, 0);
        white-space: nowrap;
        border: 0;
      }
      </style>

      <span class="visually-hidden">Accessible Text for screen readers</span>
